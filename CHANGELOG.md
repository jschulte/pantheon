# Changelog

All notable changes to this project will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [7.5.0] - 2026-03-12

### Added
- 6 standalone `.agent.yaml` files for reviewers/specialists:
  - `security.agent.yaml` — Cerberus 🔐 (independent security audit)
  - `accessibility.agent.yaml` — Iris 🌈 (WCAG compliance review)
  - `architecture.agent.yaml` — Hestia 🏛️ (structural review)
  - `logic.agent.yaml` — Apollo ⚡ (performance/logic review)
  - `quality.agent.yaml` — Arete ✨ (code quality review)
  - `pygmalion.agent.yaml` — Pygmalion 🗿 (persona forging)
- Missing `workflow.yaml` for `ux-audit` workflow
- Structural tests for `.agent.yaml` required fields and no-adapters regression

### Removed
- **BREAKING:** `src/adapters/` directory (copilot, opencode, codex, universal) — platform launchers are now auto-generated by BMAD's IDE manager from `.agent.yaml` and `workflow.yaml` files
- `scripts/detect-adapter-drift.sh` and `npm run check:drift`

### Changed
- Updated docs: README, CONTRIBUTING, PLATFORM-MIGRATION, ADR-002

## [7.4.0] - 2026-03-09

### Added
- Aletheia (Test Author) — dedicated test-generation phase (Phase 2.5 TEST) between BUILD and VERIFY
- Charon (Committer) — automated commit agent for Phase 7 COMMIT with conventional-commit formatting
- Eudaimonia (Requirements Reviewer) — requirements-coverage reviewer in Phase 3 VERIFY
- Plan-to-story skill for converting architecture plans into implementation stories
- Adversarial review mandates — reviewers must find at least one issue or justify clean bill
- Skill-based invocation model for orchestrator entry points

### Changed
- Orchestrator boundaries formalized — orchestrator routes and delegates, never performs agent work
- Phase 8 AUTO-CLOSE added for post-commit cleanup

## [7.3.1] - 2026-03-01

### Changed
- Split `<process>` block into 8 phase files under `phases/` directory (~1720 lines extracted)
- workflow.md reduced from 2163 to ~460 lines (config, routing table, footer only)
- Phase files loaded on-demand via Read tool — reduces initial context load by ~75%
- Zero changes to phase content — exact extraction from `<step>` blocks

## [7.3.0] - 2026-02-15

### Added
- Blind Reviewer Pattern: Argus reviews without builder completion artifact, preventing confirmation bias
- Conflict Resolution: Themis detects contradictory reviewer findings on same code sections with evidence-based binding judgments
- Anti-Pattern Documentation: Hermes extracts anti-patterns during reflection into formal playbook category

### Changed
- Phase 5 iteration-aware context strategy: iteration 1 resumes agents, iteration 2+ uses fresh spawns with compact context
- Orchestration Discipline: orchestrator passes data payloads + goals, not step-by-step instructions
- Phase 2/3 prompt audit: removed prescriptive step-by-step instructions, replaced with goal+context payloads

## [7.2.0] - 2026-02-01

### Added
- Persistent specialist registry at `docs/specialist-registry/` for cross-story persona reuse
- Pygmalion checks registry before forging (7-step process: Analyze, Registry, Builder, Playbooks, Gaps, Forge, Register)
- Jaccard similarity matching: REUSE (>=0.5), EVOLVE (0.3-0.49), FORGE_NEW (<0.3)
- Registry auto-initializes on first invocation (no bootstrap needed)

### Changed
- Orchestrator handles registry file I/O after Pygmalion returns
- Safety cap: max 50 specialists in registry

## [7.1.0] - 2026-01-15

### Added
- Phase 3 structural digest: orchestrator pre-reads all files once, builds ~200-400 line digest
- File classification engine: test, migration, config, route, auth, ui, database, types, security, logic
- File partitioning by reviewer concern: Argus/Hestia get all files; focused reviewers get subsets + digest
- Cache-optimized prompt structure: identical prefix cached after agent #1
- Escape hatch: focused agents can Read additional files beyond their partition
- Forged specialist review_focus keywords matched against file classifications

## [7.0.0] - 2026-01-01

### Added
- Pygmalion (Persona Forge) — dynamically forges domain-specific specialist personas
- Phase 1.5 FORGE: domain analysis + specialist forging (complexity >= light)
- Complexity gating for forged specialists: trivial/micro skip; light max 1; standard max 2; complex max 3; critical max 4

### Changed
- Phase 3 VERIFY: forged specialists spawn alongside Pantheon reviewers in parallel
- Phase 4 ASSESS: Themis triages forged specialist findings identically to Pantheon
- Phase 5 REFINE: forged specialists with upheld MUST_FIX resumed/respawned for verification

## [6.1.0] - 2025-12-15

### Changed
- Combined Mnemosyne + Hermes into single Hermes agent (saves ~5-8K tokens/story)
- Added multi-reviewer consolidated agent (saves ~60-70% Phase 3 tokens)
- Complexity-based review mode routing: consolidated for trivial-standard, parallel for complex+

## [6.0.0] - 2025-12-01

### Changed
- Renamed all agents to Greek mythology (Metis, Argus, Nemesis, Hestia, Apollo, Arete, etc.)
- Restructured to 7 named phases (PREPARE, BUILD, VERIFY, ASSESS, REFINE, COMMIT, REFLECT)
- Added Themis as independent triage arbiter (replaces builder self-triage)
- Phase numbers now show progress (e.g., 3/7)

## [5.1.0] - 2025-11-15

### Added
- 6-tier complexity scale (trivial, micro, light, standard, complex, critical)
- Pragmatic issue triage phase
- Context-aware filtering

## [5.0.0] - 2025-11-01

### Added
- Issue classification system (MUST_FIX / SHOULD_FIX / STYLE)
- Iterative refinement loop until zero MUST_FIX issues remain
- Resume original reviewers for verification
- Fresh eyes on iteration 2+
- User escalation after 3 iterations
- Tech debt logging for deferred SHOULD_FIX items

## Earlier Versions

- **v4.0** — Inspector code citations, test quality + coverage gate, playbook query + reflection
- **v3.2** — Resume Builder for fixes
- **v1.0** — Playbook Intelligence Edition: standardized playbook format, playbook index, Hermes Compaction Protocol, budget-based playbook loading, hit-rate tracking
