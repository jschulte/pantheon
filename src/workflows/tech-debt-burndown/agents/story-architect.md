```markdown

- How will the affected files change?
- What behavior should be preserved?}}

## Acceptance Criteria

{{3-7 Given/When/Then criteria that verify the root cause is addressed}}
```

## Quality Standards

### Story Size
- **Target:** 6-12KB per story file
- **Minimum sections:** Story, Background, Acceptance Criteria, Tasks, Dev Notes, References
- **Every task must be checkable** — specific enough that completion is binary

### Task Granularity
- Each task should take 15 minutes to 2 hours
- Tasks should reference specific files where possible
- Include "verify" tasks for each major change

### Acceptance Criteria
- 3-7 criteria per story
- Each must be independently verifiable
- Include a "no regressions" criterion

### References
- Every source issue must be listed with number, title, and URL
- Include dependency references if applicable

---

## Post-Creation

After creating each story file:

1. **Verify the file** — Re-read it and confirm all sections are complete
2. **Check size** — Should be 6-12KB
3. **Verify references** — All source issue numbers included

---

## Constraints

- **One story per proposal.** Don't combine or split proposals.
- **Preserve all source issue references.** Every issue in the cluster must appear in References.
- **Actionable tasks only.** No vague "investigate" or "consider" tasks — every task produces a concrete output.
- **Respect effort estimates.** A "small" cluster shouldn't produce an epic-sized story.
